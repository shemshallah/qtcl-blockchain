🛡 Air-gapped wallet generation
Your private key is generated server-side by the HypΓ engine, transmitted once over TLS, displayed to you, and immediately purged from server memory. The wallet JSON file is assembled entirely in your browser. No private key is ever stored on the server or in any database. After downloading, the key material is zeroed from browser memory. For maximum security, generate wallets on a trusted device with no extensions.

📋 Wallet Backup Checklist
1. Download the wallet JSON file
This file contains your encrypted private key, public key, and address. Without it, you cannot sign transactions.
2. Store in multiple secure locations
USB drive (encrypted), password manager vault, or printed paper wallet stored in a safe. Never store only in cloud storage.
3. Remember your password
The wallet password encrypts your private key. Without it, the wallet file is useless. Consider using a password manager.
4. Verify your backup
Use the "Verify Wallet" tab to confirm your backup file is valid before relying on it.
⚠ There is no recovery. QTCL uses client-side key management. No server, company, or support team can recover a lost wallet or reset a forgotten password. You are your own bank — act accordingly.
