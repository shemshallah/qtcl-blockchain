◈ QTCL Architecture · Quantum Cryptographic Foundation Micro-Whitepaper
§1 · HLWE Hardness Foundation

All signatures use HLWE — Hyperbolic Learning With Errors post-quantum cryptography hardened over PSL(2,ℝ) lattices. Security reduces to the Learning With Errors problem over hyperbolic space H², where exponential volume growth of balls makes standard sieving attacks collapse. Classical SVP bound: |B(r)| ~ sinh(r) ≫ e^{n/2}.

§2 · W-State Oracle Consensus

Block entropy is certified by a 5-node Byzantine oracle running a tripartite W-state density matrix (numpy, no Qiskit). Each node contributes phase_coherence measurements; fused fidelity ≥ 0.70 gates block acceptance. The Mermin M₃ value verifies genuine 3-qubit nonlocality beyond classical bound 2.0.

§3 · QRNG Ensemble

Block entropy seeds are drawn from four independent quantum sources XOR-hedged for information-theoretic security: ANU QRNG (quantum vacuum fluctuations), QBICK (ID Quantique Quantis hardware), Outshift (Palo Alto), and Random.org (atmospheric noise), then mixed via RFC 5869 HKDF.

§4 · Memory-Hard PoW

Mining is memory-hard — 4 MB SHA3-256 matrix defeating ASICs while the tessellation depth parameter scales block reward with network-proven work. Target ~60s block times at ~61k h/s gives difficulty ≈ 21-22 bits. Difficulty adjusts via EMA retarget (log₂ additive, α=0.3, window=5).
